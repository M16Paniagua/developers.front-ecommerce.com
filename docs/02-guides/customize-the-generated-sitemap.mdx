## Static Pages Customisation

You can customize the sitemap for static pages by using the `getSitemapEntries`
function in the SEO handle. You can use this function to add custom URLs to the
sitemap, set the last modification date, change frequency, and priority for each
URL.

#### Example

```tsx title="src/routes/blog.tsx"
import { createSeoHandle } from "@front-commerce/remix/seo";

export const handle = createSeoHandle({
  getSitemapEntries: ({ request, app }) => {
    return [
      {
        url: "/blog",
        lastmod: new Date(),
        changefreq: "daily",
        priority: 0.5,
        images: ["/blog-image.jpg"],
      },
    ];
  },
});
```

:::caution

If you use server-only logic within the `getSitemapEntries`, we suggest you
implement it using
[`vite-env-only`](https://remix.run/docs/en/main/discussion/server-vs-client#vite-env-only)
to ensure that the logic does not leak into the client-side code.

We have an
[existing proposal](https://github.com/remix-run/remix/discussions/9091) Remix
to implement a `serverHandle` to automatically remove code from the client
bundle.

:::

#### Opt-out of the sitemap generation for page

You can also opt-out of the sitemap generation for a specific page by returning
null or an empty array from the `getSitemapEntries` function.

```tsx title="src/routes/blog.tsx"
import { createSeoHandle } from "@front-commerce/remix/seo";

export const handle = createSeoHandle({
  getSitemapEntries: () => {
    return null;
  },
});
```
